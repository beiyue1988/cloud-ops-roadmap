# 学习路线导航

## 先选择目标

- 希望建立从零基础到中级运维的完整知识结构：进入[完整成长路线](./01-完整成长路线.md)。
- 希望先形成传统运维、云基础和 Docker 就业能力：进入[三个月就业路线](./02-三个月就业路线.md)。
- 希望判断当前能力所处层级及后续岗位方向：进入[职业发展路线](./03-职业发展路线.md)。
- 已选好路线但需要安排个人节奏：使用[每日每周学习模板](./04-每日每周学习模板.md)。
- 对某个后学章节为何依赖早期阶段有疑问：查看[知识前置依赖](./05-知识前置依赖.md)。

## 如何使用路线

1. 先在所选路线中定位阶段或章节 ID。
2. 回到对应[阶段章节清单](../知识库/README.md)查看权威章节记录。
3. 按直接前置顺序学习；不要把路线中的短说明当作章节正文。
4. 用可观察的学习证据判断是否继续，而不是只按经过的日历天数判断。

## 三类路线的关系

完整成长路线覆盖 00–20 全部阶段。十二周路线只调整优先级：主线是 291 个“必须学”章节的完整前置闭包，共 293 章；弹性集合是闭包之外的 74 个“建议学”章节；253 个“就业后补学”章节继续保留在完整成长和职业发展路线中。

十二周之后，Kubernetes、完整 CI/CD、企业日志平台、可观测性、AI Infra、AIOps/MLOps 与 SRE 仍按完整成长路线继续推进。AI Infra 与 AIOps/MLOps 是继续成长方向，不会反向加入十二周主线。

## 边界

路线不承诺固定时间掌握或就业结果，不创建正式正文、实验步骤或项目实现。项目锚点等待后续波次定义。
