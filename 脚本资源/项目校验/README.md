# 仓库校验工具

本目录提供 `cloud-ops-roadmap` 的本地 Markdown 机械门禁。工具面向 Python 3.11，仅使用标准库，无需安装依赖；它输出稳定规则编号、相对 POSIX 路径和真实数字行号，便于本地复现与解释问题。

## 运行方法

在仓库根目录执行：

```powershell
python -B "脚本资源/项目校验/validate_repository.py" .
```

位置参数 `root` 可省略，默认检查当前目录。查看命令帮助：

```powershell
python -B "脚本资源/项目校验/validate_repository.py" --help
```

工具不读取配置文件，不联网，也不修改被检查文件。

## 分级大纲门禁

默认命令仍只运行既有五类仓库校验。Phase 1 任务按交付波次显式增加 `--outline-gate`：

```powershell
python -B "脚本资源/项目校验/validate_repository.py" . --outline-gate partial
python -B "脚本资源/项目校验/validate_repository.py" . --outline-gate catalogs
python -B "脚本资源/项目校验/validate_repository.py" . --outline-gate views
python -B "脚本资源/项目校验/validate_repository.py" . --outline-gate complete
```

| 门禁 | 使用时机 | 机械责任 |
|---|---|---|
| `partial` | 单个阶段或视图任务 | 校验已经出现的受控表格、字段、章节引用和锚点语法；不要求未进入当前波次的文件存在 |
| `catalogs` | 00–20 阶段清单完成后 | 要求 21 个阶段都有合法清单，并闭合全局 ID、文件名、直接前置、依赖环和就业标签规则 |
| `views` | 波次 3 路线与索引完成后 | 包含 `catalogs`，并要求波次 3 的路线入口、`00`–`05` 视图和三类技术索引存在；校验十二周主线与 `必须学` 前置闭包 |
| `complete` | 波次 4 实践与项目映射完成后 | 包含 `views`，再要求 `06-贯穿项目演进线.md`、21 个检查点、四级实验锚点、项目入口和五个项目的双向映射闭合 |

`views` 不提前要求波次 4 才创建的 `06-贯穿项目演进线.md`；该文件在 `complete` 中成为必需项。这样各级门禁与批准实施计划的交付顺序一致。

指定大纲门禁时，既有五类校验仍会一起运行。参数只接受以上四个值；未知值由命令行参数解析直接拒绝。

## 受控 Markdown 大纲合同

每个 `知识库/NN-阶段名称/README.md` 可以在 `partial` 波次尚未开始时没有章节表。一旦出现章节表，必须位于唯一的 `## 章节清单` 下，第一张表使用精确列：

```text
章节 ID | 预定文件 | 标题 | 主要目标 | type | 直接前置 | 建议投入 | 就业标签 | 实践锚点
```

受控单元格规则：

- 章节 ID 为反引号包裹的 `NN.CC`；阶段号与目录一致，阶段内从 `01` 连续递增，全局唯一；
- 预定文件为反引号包裹的 `NN.CC-章节名称.md`，不得含空格、目录分隔符或重复文件名，并与章节 ID 对齐；
- `type` 使用反引号包裹，并从 PRD 的九种类型中选择；
- 直接前置为空时写 `—`，否则使用逗号加空格分隔的反引号章节 ID；被引用章节必须存在、排序更早，且不得自引用或形成循环；
- 建议投入只接受 `30–60 分钟`、`60–90 分钟`、`90–120 分钟`；
- 就业标签只接受 `必须学`、`建议学`、`就业后补学`；`必须学` 的递归前置不得包含 `就业后补学`；
- 实践锚点为空时写 `—`，否则使用逗号加空格分隔的反引号 `CP-NN`、`LAB-L1`–`LAB-L4` 或 `PRJ-NN-MCC`；
- 标题、主要目标和说明字段必须是单行非空文本，不接受 HTML 换行或表格竖线。

派生视图使用以下精确表头：

| 文件 | 精确表头 |
|---|---|
| `学习路线/02-三个月就业路线.md` | `周次`、`主线章节`、`弹性章节`、`阶段成果`、`实践锚点` |
| `学习路线/03-职业发展路线.md` | `能力层级`、`岗位方向`、`章节 ID`、`阶段成果`、`项目锚点` |
| `学习路线/05-知识前置依赖.md` | `后学章节`、`直接前置`、`关系说明` |
| `学习路线/06-贯穿项目演进线.md` | `项目`、`里程碑`、`所需章节`、`证据类型` |
| `学习路线/阶段检查点/README.md` | `检查点`、`阶段`、`所需章节`、`实践锚点`、`能力结果` |
| `技术索引/按技术名称.md` | `技术主词`、`别名`、`章节 ID` |
| `技术索引/按故障现象.md` | `故障现象`、`机制章节`、`排查方法章节`、`未来案例入口` |
| `技术索引/按岗位能力.md` | `岗位能力`、`章节 ID`、`阶段成果`、`项目锚点` |
| 五个项目 README | `里程碑`、`能力结果`、`所需章节`、`证据类型`、`未来内容归属` |

三个月路线必须恰好包含第 1–12 周。主线集合必须与全部 `必须学` 章节及其递归前置闭包完全相等，并按依赖拓扑排序；弹性集合只能包含 `建议学`，且不得与主线重复。

`complete` 还检查：`CP-00`–`CP-20` 只由阶段检查点表定义；`LAB-L1`–`LAB-L4` 分别由对应 Level README 定义；每个项目 README 的 `PRJ-NN-MCC` 从 `M01` 连续递增。项目表列出的章节必须反向引用里程碑，章节引用的项目里程碑也必须由项目表列回。

## 五类校验

1. **Front Matter 与章节编号：** 只有 `知识库/` 下非 `README.md` 的 Markdown 强制检查 13 个必需键、枚举、日期、受控结构和跨文件 `chapter` 唯一性。
2. **本地链接：** 全部 Markdown 检查行内文件链接、图片和引用式定义；忽略 HTTP/HTTPS、邮件与纯页内锚点。带查询或锚点的本地目标只验证文件部分。
3. **正式内容占位符：** 只有 `学习路线/`、`技术索引/`、`知识库/`、`实验手册/`、`项目实战/`、`故障案例/`、`面试体系/`、`运维思维训练/`、`企业规范/` 扫描模糊占位词及模板标记。
4. **高置信度敏感信息：** 全部 Markdown 扫描固定结构云 AccessKey、私钥头和敏感字段直接赋值；诊断只报告类型和位置，不回显匹配值。
5. **仓库与输入输出：** 排除 `.git/`，以 UTF-8 读取 Markdown，聚合确定性诊断和扫描统计；根目录、读取以及根路径解析或遍历失败转换为 `IO` 规则，不打印 traceback。

`模板/`、`项目管理/`、`项目记忆/`、`版本记录/`、`参考资料/`、`脚本资源/` 和根治理文件不执行占位符规则，但仍执行链接与敏感信息检查。

## 受控语法边界

Front Matter 不是完整 YAML 解析器，只接受：

- 文件第一行与独立结束行的 `---`；
- 顶层 `key: value`；
- `prerequisites`、`objectives`、`keywords` 的一层缩进非空列表；
- `versions: {}` 或 `versions` 下的一层非空映射；
- 普通标量、合法成对的单引号或双引号标量，以及空行和注释行。

只有开头或结尾引号、首尾引号不同、闭合引号后存在尾随内容，以及约定外的复杂引号都会被 `FM008` 拒绝。本工具不实现通用 YAML 引号转义。锚点、别名、标签、折叠/多行标量、复杂键和更深嵌套同样会被 `FM008` 拒绝。

链接识别是项目约定的 Markdown 子集，不是完整 CommonMark 解析器。它支持行内链接、图片、引用式定义和尖括号目标；复杂目标中如有嵌套括号，应改用尖括号形式。链接和占位符忽略围栏代码，链接还忽略行内代码。敏感信息扫描包含代码围栏，避免示例凭据进入仓库。

相对链接必须留在仓库根目录内。绝对路径、Windows 盘符、越界路径、缺失目标和实际文件名大小写不一致会分别报告；即使当前 Windows 文件系统不区分大小写，也按磁盘实际名称检查，以避免在 Linux、GitHub 或文档站失效。

## 规则编号

| 前缀 | 编号 | 含义 |
|---|---|---|
| `FM` | `FM001`–`FM008` | Front Matter 边界、必需键、重复键、空值、枚举、日期和受控结构 |
| `CH` | `CH001` | 重复 `chapter`，每个冲突文件分别报错 |
| `LINK` | `LINK001`–`LINK005` | 缺失、越界、绝对路径/盘符、大小写和无法解析的目标 |
| `PH` | `PH001`–`PH004` | 英文/中文模糊占位符和模板填写标记 |
| `SEC` | `SEC001`–`SEC003` | 固定结构 AccessKey、私钥头和敏感字段直接赋值 |
| `IO` | `IO001`–`IO003` | 根目录缺失/非目录、Markdown UTF-8 读取、根路径解析或遍历错误 |
| `OL` | `OL001`–`OL012` | Phase 1 章节清单、依赖、路线、索引、锚点和项目映射 |

大纲诊断的固定职责如下：

| 编号 | 固定责任 |
|---|---|
| `OL001` | 当前门禁缺少规定文件、标题或表格 |
| `OL002` | 表头、列数、分隔或受控语法错误 |
| `OL003` | 章节 ID 格式、阶段号、连续性或唯一性错误 |
| `OL004` | 预定文件格式、ID 对齐或唯一性错误 |
| `OL005` | 标题、目标、`type`、投入或就业标签非法 |
| `OL006` | 前置缺失、自引用或前向引用 |
| `OL007` | 依赖循环 |
| `OL008` | 路线或索引引用不存在，或路线顺序违反前置 |
| `OL009` | 十二周集合、前置闭包或就业标签冲突 |
| `OL010` | 锚点格式、定义位置、连续性或引用不存在 |
| `OL011` | 章节与项目里程碑双向映射不一致 |
| `OL012` | 派生视图重复标识或越界集合成员 |

诊断格式为：

```text
相对/路径.md:行号: 规则编号 错误说明
```

根目录错误使用 `.:0`。诊断按路径、数字行号、规则编号和消息排序；敏感信息错误不会包含命中原文。

`IO001` 表示根路径不存在或不是目录；`IO002` 表示单个 Markdown 无法按 UTF-8 读取；`IO003` 表示根路径规范化、目录发现或相关遍历触发了预期的路径/文件系统错误。`IO003` 使用固定脱敏消息，不回显根路径或底层异常文本。

## 输出与退出码

校验通过时向标准输出写入 Markdown 数、知识章节数、本地链接数和零错误摘要，并返回 `0`：

```text
校验通过：Markdown 68，知识章节 0，本地链接 39；错误 0
```

数字以实际仓库为准。校验失败时向标准错误逐行输出诊断及错误总数，并返回 `1`。`--help` 返回 `0`。同一仓库状态下连续运行的合并输出和退出码应完全一致。

大纲门禁通过时，在原有摘要之后追加一行，不改变默认摘要字段和顺序：

```text
大纲门禁通过：级别 partial，阶段清单 0，章节 0，依赖 0，视图引用 0，锚点 0；错误 0
```

严格门禁失败时继续使用同一诊断格式与退出码 `1`，不会打印 traceback、绝对个人路径或敏感值。

## 运行测试

在仓库根目录执行完整标准库测试：

```powershell
python -B -m unittest discover -s "脚本资源/项目校验/tests" -p "test_*.py" -v
```

测试使用临时目录构造正反例，不安装 `pytest`、PyYAML 或 Markdown 解析器。

## 明确限制

- 不自动修复文件；
- 不访问或判断外链可达性；
- 不验证页内锚点是否存在或正确；
- 不支持完整 YAML、完整 Markdown 或完整 CommonMark；
- 大纲表解析只支持本文列出的受控 Markdown 表格，不是通用 Markdown 表格解析器；
- 工具只验证可机械证明的结构、引用、集合和双向映射，不决定实际章节标题、颗粒度或教学质量；
- 高置信度敏感信息规则不是安全审计，也不证明仓库绝对没有 Secret；
- 不检查技术真实性、教学质量、来源权威性或版本兼容性；
- 不替代人工审查、独立审查或最终验收；
- P0-04 不接入 CI。未来 CI 如获单独授权，应调用同一 CLI，不能复制一套规则。
