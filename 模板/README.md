# 模板使用说明

本目录是内容与治理交付物的统一复制入口。模板只提供结构、字段和填写边界，不替代 `CODEX-PRD.md`、`AI-CODEX-RULES.md`、`AGENTS.md` 或具体任务包，也不包含正式教学答案、实验结果或动态版本结论。

README 不计入模板数量；本目录应有十类模板、共十一个 Markdown 文件。

## 复制方法

1. 复制目标模板到任务包授权的目标路径，不要在本目录直接生产正式内容。
2. 知识章节模板保留并填写 YAML Front Matter；其他模板使用 Markdown 字段。
3. 用实际内容替换所有占位符。填写说明使用 `<!-- -->` HTML 注释，理解后必须从正式交付物中删除。
4. 不适用的可选模块必须连同标题、说明和占位符整段删除；不得保留空标题。
5. 按任务包运行字段、链接、版本、安全、格式和 Git 范围复验。

## 占位符协议

只允许以下两类占位符：

- `<必填：说明>`：复制后必须替换为实际内容，不能删除字段或模块；
- `<可选：说明>`：根据学习目标和交付物类型填写；不适用时整段删除。Front Matter 固定键不能删除时，按字段说明写入明确的空值或约定值。

YAML 值中的占位符必须加引号。正式内容、实验、项目、故障、面试、图表说明和治理交付物不得残留占位符、HTML 填写说明或空标题。禁止使用模糊占位词替代这两类协议。

## 职责层级

- `AGENTS.md`：角色、权限、文件所有权与验收责任；
- `CODEX-PRD.md`：产品目标、范围、信息架构、字段和质量目标；
- `AI-CODEX-RULES.md`：跨内容研究、写作、安全、核验与证据规则；
- 本目录模板：单类交付物的稳定结构，不扩展为第二份规格；
- 当前任务包：本次唯一目标、白名单、验证、提交和停止条件。

下层文件可以收紧当前任务要求，但不得放宽上层安全、质量或权限边界。发现冲突时停止写作并交由主会话裁决。

## 版本、来源、证据与安全

- 产品版本只通过 PRD、已批准 ADR 或现行版本入口核定；模板不得写死动态补丁版本。
- 来源优先使用官方文档、标准、RFC、上游仓库、发布说明或厂商一手资料，并记录核验日期和适用范围。
- 证据声明区分来源核验、静态检查、环境验证、综合实验、生产证据和未验证；一种证据不能冒充另一种。
- 示例输出必须标明“示意输出”；只有实际执行且可追溯的结果才能称为“实际输出”或“已验证”。
- 禁止保存密码、Token、Cookie、私钥、云 AccessKey、真实个人信息或未经脱敏的生产证据。

## 模板变更复验

模板新增、删减或修改字段后，必须重新检查十类模板的文件集合、占位符、Front Matter、关键字段、相对链接、硬编码版本、敏感信息、空白和职责边界。模板变更只能由获得相应白名单的任务执行，执行会话不得自行改变 PRD、AI 规则、版本基线或任务状态。
