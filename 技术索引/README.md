# 技术索引

## 目录职责

本目录把 21 个阶段的权威章节清单组织成三类查询视图，帮助读者按技术、可观察故障现象或岗位能力定位学习入口。索引只提供稳定章节 ID 和必要短说明，不成为第二份章节目录或知识正文。

章节 ID、标题、`type` 与直接前置以[知识库](../知识库/README.md)及各阶段 README 为权威来源；学习先后和能力递进可结合[学习路线](../学习路线/README.md)判断。

## 索引入口

- [按技术名称](./按技术名称.md)：按中英文技术主词、缩写和常用称呼查找章节。
- [按故障现象](./按故障现象.md)：从十类可观察现象进入机制与排查方法章节。
- [按岗位能力](./按岗位能力.md)：按职业发展路线冻结的七个能力层级定位代表章节。

## 使用边界

- 技术名称索引覆盖当前阶段清单中 `tool`、`service`、`platform`、`language`、`cloud-service`、`hardware` 六类全部 146 章；额外机制章节只用于补足查询上下文。
- 故障现象索引只连接机制与排查方法，不提供处置步骤；独立故障案例由后续任务维护。
- 岗位能力索引消费学习路线的固定七层，不把项目锚点或未决资源组合作为能力成立条件。
- 阶段清单或职业路线经批准变化时，必须从权威输入重新核对并原子更新受影响索引。
