# Level 4 架构实验

`LAB-L4` 定义架构设计与取舍：学习者根据明确需求和约束比较方案，说明容量、可靠性、安全、成本与可维护性之间的权衡，并规划验证或演练。

## 准入

- 已具备相关系统的部署、运行维护与故障恢复能力；
- 能提供需求、负载、风险、恢复目标、权限和成本等设计输入；
- 能区分已验证事实、设计假设与仍需验证的条件。

## 能力边界

本级证明方案分析和工程决策能力，不自动证明方案已在生产运行。具体产品版本、云资源、GPU、模型或价格等动态选择，必须由对应任务的版本与成本门另行核定。

## 退出条件

设计记录能够追溯需求与约束，至少比较可行方案并解释取舍，列明失效边界、验证门、演练计划和退出条件。

## 风险边界

不得把未核验假设写成运行事实，也不得因完成设计而执行生产变更、购买资源或扩大权限。成本、数据、安全或恢复条件不清时，应保留为未决门。

## 证据

允许的证据类型为设计记录、权衡说明和演练方案。证据应明确假设、适用范围和验证状态，不能用架构图或方案名称替代论证。
