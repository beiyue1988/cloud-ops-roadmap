# Level 3 企业实验

`LAB-L3` 定义企业运行与故障处理：学习者在多节点或等效企业运行边界内维护系统，依据证据定位受控故障，并验证恢复结果。

## 准入

- 已能部署并验证实验涉及的组件组合；
- 能识别节点、依赖、数据、访问与服务影响范围；
- 已建立基线状态、观测入口、故障停止条件和恢复路径。

## 能力边界

允许多节点运行维护以及可观察、可控、可恢复的故障。未授权生产系统、真实业务数据、不可逆破坏和缺少恢复路径的故障注入不属于本级范围；容量与架构选型也不以本级实验裁决。

## 退出条件

学习者能够以证据缩小故障范围，区分临时恢复与根因处理，并证明系统恢复到约定状态且无遗留影响。

## 风险边界

故障必须限定对象、持续时间与最大影响范围。无法持续观察、达到停止条件、恢复验证失败或爆炸半径扩大时，应立即终止并执行恢复。

## 证据

允许的证据类型为运行证据、故障定位和恢复验证。证据必须保留事件顺序与关键状态，不得把重启成功等同于根因关闭。
