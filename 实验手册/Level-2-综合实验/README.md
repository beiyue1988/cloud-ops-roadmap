# Level 2 综合实验

`LAB-L2` 定义多知识点组合：学习者在单主机或其他受控范围内连接多个组件或能力，形成可验证的部署结果，并保留依赖与回滚证据。

## 准入

- 已具备实验所列各知识点的单项操作与验证能力；
- 能说明组件之间的依赖、输入、输出和预期状态；
- 已确定部署范围、恢复基线和清理责任。

## 能力边界

允许多个知识点和组件，但保持单主机或等效受控范围。多节点企业运行、以故障恢复为主要目标的维护活动以及架构级方案比较不属于本级完成范围。

## 退出条件

组合结果达到可观察的完成标准，关键依赖均已验证，并且能够按预定回滚路径恢复基线。

## 风险边界

开始前必须确认组件影响范围、数据保留和依赖顺序。出现依赖来源不明、回滚无法验证、真实业务暴露或资源成本无法界定时停止。

## 证据

允许的证据类型为部署验证、依赖验证和回滚记录。证据应能区分单项成功、组合成功与回滚成功，不能用其中一项替代其他项。
