# 实验手册

## 目录职责

本目录按实践复杂度组织四级实验入口。等级表示学习者需要组合多少知识、承担多大风险以及提交什么证据，不表示章节难度，也不替代知识正文。

## 等级入口

- [Level 1 基础实验](./Level-1-基础实验/README.md)：观察并验证单个知识点；
- [Level 2 综合实验](./Level-2-综合实验/README.md)：组合多个知识点完成受控部署；
- [Level 3 企业实验](./Level-3-企业实验/README.md)：维护多节点系统并处理受控故障；
- [Level 4 架构实验](./Level-4-架构实验/README.md)：围绕容量、可靠性、安全和成本比较方案。

学习者应以目标实验声明的准入条件为准，不因目录顺序自动取得下一等级资格。每一级的完成都必须由对应入口规定的退出条件和证据判断。

## 权威关系

- [知识库](../知识库/README.md)提供原理与前置章节；
- [阶段检查点](../学习路线/阶段检查点/README.md)把每个阶段的末章、实验深度和能力结果连接起来；
- 未来实验正文以所属 Level 目录为唯一归属，并引用知识章节，不复制知识正文。

## 当前边界

当前只建立等级合同和导航，不登记实验文件，不提供实验步骤、命令、配置、故障注入、运行结果或项目实现。未来实验仍须按[实验模板](../模板/实验模板.md)说明环境、风险、回滚、验证和清理。
