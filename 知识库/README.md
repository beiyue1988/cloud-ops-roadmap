# 知识库

## 目录职责

本目录按 00 至 20 的成长阶段组织知识章节。Phase 1 已形成完整的文件级章节目录：21 个阶段清单共登记 620 章和 817 条直接前置边，为学习路线、技术索引、实践映射和后续单文件任务提供稳定引用目标。

以上统计来自已验收的 21 个阶段清单，只用于说明当前目录规模，不替代各阶段 README 中的权威章节表。

## 权威内容类型

`知识库/` 是正式知识正文的唯一权威来源。概念、原理、平台、服务、工具、语言、云服务、硬件和方法论正文只在对应阶段维护一份。

每个 `知识库/NN-阶段名称/README.md` 是该阶段章节目录的权威来源，负责章节顺序、章节 ID、预定文件、标题、主要目标、`type`、直接前置、建议投入、就业标签和实践锚点。学习路线与技术索引是从这些清单派生的视图，只引用稳定章节 ID，不成为第二套章节目录或知识正文。

## 非范围

- 不复制学习路线或技术索引；
- 不保存实验步骤、项目实现、独立故障案例或面试题；
- 不在总入口复制任一阶段的章节表或正式知识正文；
- Phase 1 不创建任何正式知识章节。

## 命名规则

- 阶段目录使用 `NN-阶段名称`，编号范围为 00 至 20；
- 阶段入口固定为 `README.md`；
- 技术名保留英文，名称不含空格并以短横线连接；
- 后续章节文件的细化命名不在本任务定义。

## 章节清单合同

各阶段 README 的章节清单统一包含九列：

| 字段 | 含义与约束 |
|---|---|
| 章节 ID | 使用与阶段一致的 `NN.CC` 稳定标识，阶段内连续编号。 |
| 预定文件 | 记录未来单章 Markdown 的唯一文件名，并与章节 ID 对齐。 |
| 标题 | 用可理解的短标题准确限定单章范围。 |
| 主要目标 | 每章只记录一个可检查的主要教学目标。 |
| `type` | 从 PRD 批准的九种知识类型中选择，用于声明内容性质。 |
| 直接前置 | 只记录完成本章必须先掌握的最小必要边，不重复传递性前置。 |
| 建议投入 | 使用 30–60、60–90 或 90–120 分钟三档建议，不承诺掌握时长。 |
| 就业标签 | 在 `必须学`、`建议学`、`就业后补学` 中选择，作为路线派生输入。 |
| 实践锚点 | 仅把章节映射到阶段检查点、实验等级或项目里程碑，不展开实践内容，也不替代实践侧定义。 |

章节直接前置共同构成依赖图；就业主线由就业标签和全部必需前置的递归闭包派生，而不是另行维护一份章节清单。仅有帮助但不是完成本章所必需的内容，不进入直接前置。

## 维护合同

- Phase 1 验收后的章节 ID 保持稳定，不得把旧 ID 静默复用于无关主题；
- 章节确需重排、改名、增删或改变依赖时，必须通过独立维护任务，原子同步对应阶段清单及学习路线、技术索引、依赖和实践映射等派生视图；
- 发现阶段清单与派生视图不一致时，以已验收的阶段 README 为章节目录权威来源，并停止在派生视图中顺带修订源清单；
- 自动化只验证结构、引用、集合和映射的一致性，不生成章节、导航或正文，也不引入中央注册表作为第二数据源。
