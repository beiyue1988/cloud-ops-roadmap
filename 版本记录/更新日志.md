# 版本基线更新日志

## 用途与规则

本文件追踪 `软件版本基线.md` 的建立、升级、降级、回退和证据状态变化。记录只追加，不覆盖或删除历史；错误记录以新条目纠正并链接原条目。外部 Release Notes 只作为证据链接，不复制到本文件。

major、minor、固定 tag 或架构组合变化必须由新 ADR 或替代 ADR 批准。仅 patch 变化且兼容边界未改变时，可按基线流程更新，但必须保留来源核验与环境验证状态。

## 变更记录字段

每条记录必须包含：日期、组件、旧值、新值、变更类型、原因、证据、验证状态、影响范围、关联任务/ADR、回退说明。
