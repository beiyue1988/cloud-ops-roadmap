# 企业规范

## 目录职责

本目录组织可复用的企业运维治理规则与工程约定。

## 权威内容类型

未来主机、账号、权限、配置、日志、备份、制品、变更、发布、云资源和 AI 资源治理规范以本目录为权威归属。

## 非范围

- 不复制知识正文、实验步骤、项目实现或故障案例；
- 不保存真实密码、Token、Cookie、私钥或云 AccessKey；
- 本任务不创建规范正文、清单或模板。

## 命名规则

- 目录入口固定为 `README.md`；
- 后续文件名按治理领域组织，不含空格并以短横线连接，技术名保留英文。

## 上游与下游

- 知识来源：[知识库](../知识库/README.md)
- 工程实践：[项目实战](../项目实战/README.md)
- 思维训练：[运维思维训练](../运维思维训练/README.md)
- 版本约束：[版本记录](../版本记录/README.md)
