# Phase 0 治理与仓库骨架实现计划

> **面向独立执行会话：** 本计划由主会话制定。每个 `P0-xx` 任务必须在独立新会话中执行；执行会话只完成自己领取的任务，不连续执行后续任务。步骤使用复选框跟踪，由主会话在验收后更新项目状态。

**目标：** 建立一个可供多个独立会话安全协作、可验证、可追踪的知识库治理与仓库骨架，不生产任何正式学习章节。

**架构：** `AGENTS.md` 与 `CODEX-PRD.md` 是全局规则源；`项目管理/` 保存任务、计划、决策和验收；`项目记忆/` 提供简短索引；内容目录使用 README 声明所有权；模板定义交付结构；标准库 Python 校验器提供自动化门禁。

**技术栈：** Markdown、YAML Front Matter、PowerShell、Python 3 标准库、Git。

---

## 执行规则

1. 新会话开始时必须阅读根目录 `AGENTS.md`、`CODEX-PRD.md`、本计划和对应任务包。
2. 每个任务只修改“文件”清单列出的路径。
3. `AGENTS.md`、`CODEX-PRD.md` 和本计划由主会话拥有，执行会话不得修改。
4. 一个任务提交后，后续任务必须等待主会话将前置任务标记为 `accepted`。
5. 执行会话必须提供实际命令、退出码和关键输出；没有验证证据不能报告完成。
6. 当前 `.git` 目录为空且尚不是有效仓库。只有 `P0-01` 被授权初始化 Git；不得删除或移动现有 `.git`。
7. 所有文件使用 UTF-8，目录按 PRD 使用中文，技术名和机器字段保留英文。
