# P0-05 版本与来源基线实施计划

> **面向 AI 代理的工作者：** 必需子技能：使用 `executing-plans` 按任务顺序执行本计划。步骤使用复选框（`- [ ]`）跟踪进度；不得委派、跳步或自行扩大白名单。

**目标：** 建立可追溯、可升级且不提前冻结全部技术栈的软件版本与一手来源基线。

**技术栈：** Markdown、官方/上游一手资料、现有 Python 3.11 仓库校验器、PowerShell、Git。

## 2. 固定语义与数据约束

### 2.1 治理状态

只能使用：

```text
locked
phase-gated
reference-only
```

- `locked` 必须同时写明锁定粒度；
- `phase-gated` 必须写明最晚核定节点，版本字段不得填猜测值；
- `reference-only` 必须写明为什么不能作为默认教学线。

### 2.2 证据状态

只能使用：

```text
source-verified
environment-verified
not-yet-verified
```

- P0-05 浏览官方资料只能产生 `source-verified`；
- 仓库校验器 Python 3.11.15 可依据 P0-04 验收记录为 `environment-verified`；
- 其他组件不得在没有既存环境证据时写 `environment-verified`；
- 阶段门组件默认使用 `not-yet-verified`。

### 2.3 主表字段

`版本记录/软件版本基线.md` 的主表列固定为：

```text
组件或组件组
用途范围
教学线或固定标识
当前来源核验版本
锁定粒度
治理状态
证据状态
选择理由
最晚核定或验证节点
官方来源
核验日期
更新触发条件
```

可以在表后增加解释，不得删除、合并或重命名这些列。

## 13. 停止条件

出现以下任一情况立即停止，不得用替代版本或模糊措辞继续：

1. 启动状态、分支、远程、Python 或工作区不符；
2. 任一创建目标已存在或必需 README 缺失；
3. P0-05 不是 `ready`；
4. 设计、计划、任务包、PRD、AI 规则、ADR 模板或版本事实冲突；
5. Kubernetes 1.36 已不受支持，或 containerd/Calico 兼容交集不成立；
6. RuoYi-Vue tag、commit、仓库或许可证无法由官方来源确认；
7. 需要选择未批准的 minor、major、tag、平台、许可证或 GPU 组合；
8. 关键事实只能由搜索摘要、第三方文章或 AI 文本支持；
9. 来源冲突无法解释；
10. 需要白名单外修改，或仓库校验失败根因位于白名单外；
11. 发现疑似真实凭据、签名 URL 或个人信息；
12. 无法保证两个提交范围，或同一外部阻塞连续出现三次。
