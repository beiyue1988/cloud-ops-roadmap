# P0-03 内容与治理模板实施计划

> **面向 AI 代理的工作者：** P0-03 执行会话必须使用 `executing-plans` 技能逐任务实施本计划，并用复选框跟踪进度；提交前必须使用 `verification-before-completion`。项目要求所有模板生产交给独立新会话，主会话只签发任务、审查和验收。

**目标：** 创建十类可直接复制的内容与治理模板及其索引，并通过独立审查形成后续内容生产的稳定模板基线。

**架构：** `模板/README.md` 负责统一入口与占位符协议；十个模板文件各自承担一种交付物结构，不复制 PRD 或 AI 规则；P0-03 执行、P0-03B 独立审查和主会话验收依次串行。模板只提供结构和说明，不包含正式教学答案、实验结果或动态版本。

**技术栈：** Markdown、YAML Front Matter、HTML 注释、PowerShell、Git。

**批准设计：** [P0-03 内容与治理模板设计说明](./2026-07-11-P0-03-内容与治理模板-设计说明.md)，设计提交 `04cb2aea60a8914ae1a73cdfd3e7983af37cc577`。

## 2. 执行顺序与提交边界

1. 主会话签发 P0-03 任务包并把 P0-03 更新为 `ready`；
2. 独立执行会话创建 11 个模板目录文件；
3. 执行会话完成字段、占位符、链接、安全和 Git 检查；
4. 执行会话提交模板：`docs: add project content templates`；
5. 执行会话创建并单独提交报告：`docs: add P0-03 execution report`；
6. 主会话完成范围初审并解锁 P0-03B；
7. 独立审查会话提交：`docs: review project content templates`；
8. 主会话形成验收报告和状态决定；
9. P0-03 `accepted` 后，P0-04 才能进入任务包制定流程。

任何会话都不得 amend、rebase 或重建已经提交并被其他报告引用的提交。

- [ ] **步骤 4：运行局部检查**

```powershell
$knowledge = '模板\知识章节模板.md'
$keys = @('title','chapter','stage','type','difficulty','prerequisites','objectives','study_time','practice_level','keywords','versions','status','last_updated')
foreach ($key in $keys) {
  if (-not (Select-String -LiteralPath $knowledge -Pattern "^${key}:")) { Write-Error "缺少 $key"; exit 1 }
}
$types = @('concept','principle','service','platform','tool','language','cloud-service','hardware','methodology')
foreach ($type in $types) {
  if (-not (Select-String -LiteralPath $knowledge -SimpleMatch -Pattern $type)) { Write-Error "缺少 type $type"; exit 1 }
}
$statuses = @('draft','reviewed','verified','deprecated')
foreach ($status in $statuses) {
  if (-not (Select-String -LiteralPath $knowledge -SimpleMatch -Pattern $status)) { Write-Error "缺少 status $status"; exit 1 }
}
$core = @('本章目标与前置知识','技术定位','核心概念或关键规则','原理、流程或使用模型','实际场景或实践','验证与常见问题','学习检查','参考资料')
foreach ($heading in $core) {
  if (-not (Select-String -LiteralPath $knowledge -Pattern "^## .*${heading}")) { Write-Error "缺少核心标题 $heading"; exit 1 }
}
```

## 3. 停止条件

执行或审查会话遇到以下任一情况立即停止：

- 启动时工作区不干净、分支或远程不符；
- 目标文件已经存在；
- 任务状态或前置验收不符；
- 需要修改白名单外文件；
- PRD、AI 规则、批准设计和任务包存在无法解释的冲突；
- 需要新增第十一类模板、改变 Front Matter、状态、阶段、项目或版本基线；
- 模板无法在当前范围内同时满足直接复制和机械校验；
- 发现敏感信息或未经授权的真实凭据；
- 自动验证无法在白名单内修复；
- 同一外部阻塞连续出现三次。

## 4. 完成定义

只有同时满足以下条件，P0-03 才可标记 `accepted`：

1. 十类模板、README 和执行报告均由独立执行会话创建；
2. 模板目录精确 11 个 Markdown 文件；
3. 知识章节 Front Matter 13 个键、九种 `type`、四种内容状态完整；
4. 十类模板满足批准设计的字段和语义要求；
5. 占位符只使用批准格式，模糊占位词为 0；
6. 硬编码动态版本、敏感信息、失效链接和空白错误均为 0；
7. 模板和执行报告形成两个规定提交；
8. P0-03B 由未参与创建的会话独立审查；
9. 独立审查无阻塞、无必须修改；
10. 主会话完成独立复验和验收报告；
11. 全过程未执行未经授权的远程写操作；
12. 最终工作区干净。
