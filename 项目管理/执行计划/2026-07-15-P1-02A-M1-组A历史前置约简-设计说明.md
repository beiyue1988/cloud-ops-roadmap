# P1-02A-M1 组 A 历史直接前置约简设计说明

## 1. 状态与目的

- **状态：** 方案已获用户批准，待书面规格复核
- **设计日期：** 2026-07-15
- **所属范围：** Phase 1 波次 2，组 A 集成前维护
- **前置基线：** P1-02-04/R1 已于提交 `5b8c8379df6c3de76f27e03fe9fcb7fcc0d1da83` 完成主会话验收

P1-02-04 的累计图审计发现，已接受的阶段 01/02 中仍有四条直接前置存在替代可达路径。它们不影响图的无环性，也不阻塞阶段 04 的验收，但会使组 A 的依赖图保留非最小表达，并在后续阶段审计中反复形成已知噪声。

本设计选择在 P1-02-05 前签发一个独立维护任务，精确删除四条历史传递冗余边，经独立复审和主会话验收后再继续组 A 最后一个阶段。主会话只负责规格、任务签发与验收，不直接修改阶段产品。

## 2. 方案比较与裁决

| 方案 | 优点 | 代价与风险 | 裁决 |
|---|---|---|---|
| 立即创建独立维护任务 | P1-02-05 从干净累计图开始；`P1-02A-R` 只承担审查职责；问题和提交范围最小 | 在 P1-02-05 前增加一次执行与验收 | **采用** |
| P1-02-05 后、`P1-02A-R` 前维护 | 不打断阶段编号推进 | 阶段 05 会再次携带和披露四条已知冗余，增加累计图审计噪声 | 不采用 |
| 由 `P1-02A-R` 顺带修改 | 少一个任务编号 | 混淆审查与实施，违反规格、执行和验收分离 | 禁止 |

## 3. 精确产品合同

任务编号固定为 `P1-02A-M1`，名称为“约简阶段 01/02 历史直接前置”。产品只允许修改：

```text
知识库/01-计算机与服务器基础/README.md
知识库/02-网络基础/README.md
```

边集合差异必须精确为四删、零新增：

| 删除的直接边 | 已存在的替代路径 | 删除后的直接前置单元格 |
|---|---|---|
| `01.16 -> 01.03` | `01.16 -> 01.05 -> 01.03` | `01.16: 01.05, 01.08, 01.10, 01.12, 01.14` |
| `02.14 -> 02.02` | `02.14 -> 02.04 -> 02.02` | `02.14: 02.04` |
| `02.21 -> 02.15` | `02.21 -> 02.20 -> 02.15` | `02.21: 02.20` |
| `02.23 -> 02.20` | `02.23 -> 02.22 -> 02.21 -> 02.20` | `02.23: 02.11, 02.22` |

除上述四个“直接前置”单元格外，两份 README 的其余八列、全部其他章节行、全部表外文本必须逐字不变。不得借维护任务重命名章节、调整标题或目标、改变类型/投入/就业标签/实践锚点，也不得修改阶段 00、03、04 或未来阶段。

- P1-02-01、P1-02-02 原执行报告保持历史原样，由新的维护报告记录替代后的图统计。

本任务只约简依赖表达，不改变任何章节的学习范围、先后可达关系或原验收结论。删除直接边后，四组前置关系仍分别通过表中替代路径成立。

## 5. 执行、报告与审查边界

新的独立执行会话负责两份产品 README 和自己的执行报告：

```text
项目管理/执行报告/P1-02A-M1-执行报告.md
```

产品与报告必须分成两个本地提交：产品提交只含两份 README，报告提交只含维护报告。执行会话只能建议 `submitted`，不得更新任务看板或宣布 `accepted`。

由于任务跨越两个已接受阶段，产品提交后必须由新的独立审查会话检查：

1. 精确四删、零新增和其他字段冻结；
2. 四条替代路径在提交后仍真实存在；
3. 00–04 累计图无环且传递冗余为 0；
4. 原报告未被篡改，新报告没有改写历史；
5. 没有正式正文、跨阶段扩写、脚本、配置、实验或远程写操作。

独立审查通过后，主会话重新运行全部门禁并作最终裁决。只有 `P1-02A-M1` 被主会话接受，才签发 P1-02-05；`P1-02A-R` 仍必须等待 P1-02-05 完成。

## 7. 停止条件

出现以下任一情况必须停止，不得自行扩大修订：

- 任一指定边不存在或替代路径不成立；
- 删除后出现缺失引用、前向引用、环或新的传递冗余；
- 需要新增、替换或重排其他前置边；
- 需要改变章节标题、目标、标签、锚点或其他已接受字段；
- 测试、默认校验、partial 或格式检查失败且不能在白名单内解释；
- 工作区有未知修改，或需要修改任务看板、原执行报告、PRD、ADR、版本基线或其他治理基线；
- 需要 push、Pull Request、发布、凭据、安全控制或其他外部写操作。

发生停止条件时，执行会话只保留证据并交回主会话裁决。
