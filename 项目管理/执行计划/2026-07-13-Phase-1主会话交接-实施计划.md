# Phase 1 主会话交接实施计划

> **面向 AI 代理的工作者：** 必需子技能：使用 `executing-plans` 按任务顺序执行本计划。步骤使用复选框（`- [ ]`）跟踪进度。本计划属于主会话治理工作，只能由当前 Phase 0 主会话执行，不得委派给内容执行会话或独立审查会话。

**目标：** 建立可版本化的 Phase 1 主会话交接入口和完整启动指令，使新的唯一主会话能够在不依赖旧聊天记录的情况下接管 Phase 1 治理。

**架构：** 新增专用会话交接目录及职责 README，把权限转移、Phase 0 状态、必读路由、启动门禁、Phase 1 边界和复制指令集中到单一交接文件；只在 `项目管理/README.md` 增加入口。交接文件不复制权威规范全文，不预测自身提交哈希，新会话通过 Git 查询取得交接提交。

**技术栈：** Markdown、PowerShell、Git、Python 3.11.15、现有仓库校验器。

## 10. 失败与停止条件

任一情况发生时停止，不提交交接成果：

- 三文件之外出现修改；
- Phase 0 最终验收提交或设计提交不存在/不在当前历史；
- 44 项测试、仓库校验、链接、格式或正式内容空边界失败；
- 启动指令缺少角色、目录、必读、目标、边界、验证、停止条件或首轮回复；
- 交接文件错误地把 Phase 1 描述为已授权批量正文；
- 存在敏感信息、远程写授权或两个主会话并行裁决；
- 需要修改 ADR、版本基线、任务看板或全局规则才能完成交接；
- 用户在提交前撤回或修改交接决定。
