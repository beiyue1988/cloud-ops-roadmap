# ADR-0002：双许可证策略

- 状态：`accepted`
- 日期：2026-07-10

## 背景

本仓库同时包含教学内容、原创图表、脚本、配置和代码。内容共享与代码复用需要不同的许可模型；贯穿项目还会使用 RuoYi-Vue 等第三方材料，必须保留原许可证和归属。

## 决定

1. 原创 Markdown 正文、学习说明、实验与项目说明以及原创图表采用 CC BY-SA 4.0。
2. 原创脚本、配置、代码文件和明确标记的代码片段采用 Apache License 2.0。
3. 根目录 `LICENSE` 说明范围，`LICENSE-CONTENT` 引用 CC BY-SA 4.0 官方许可，`LICENSE-CODE` 保存 Apache License 2.0 官方文本。
4. 第三方内容保持原许可证，不被本项目再许可；RuoYi-Vue v3.9.2 保持 MIT License。
5. 未来新增任何第三方代码、图片、文档或素材，都必须在 `THIRD-PARTY-NOTICES.md` 或材料自身声明中登记来源、版本、版权和许可证。

## 后果

学习内容可以在署名和相同方式共享的条件下传播，代码可以按 Apache License 2.0 复用，同时第三方权利边界保持清晰。贡献者需要判断材料类型，并在混合文件或第三方材料中增加明确许可标记；下游使用者也必须分别遵守适用许可。

## 复审触发条件

当内容与代码边界无法可靠区分、计划引入不兼容许可证、发布形态改变许可义务，或法律与组织要求发生实质变化时，由主会话复审。
