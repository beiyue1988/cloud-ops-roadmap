# 决策记录

本目录保存影响范围、架构、许可、版本或治理方式的正式 Architecture Decision Record（ADR）。文件编号使用连续四位序号：`ADR-0001-简短标题.md`。

## 允许编辑者

主会话负责创建、接受、替代和废弃 ADR；执行会话只有在任务包列出精确 ADR 路径时才可创建草案或已批准决策的记录，不得自行改变全局规则。

## 必需字段

每份 ADR 必须包含编号与标题、状态、日期、背景、决定、后果和复审触发条件。涉及外部事实时还要记录一手来源和核验日期。

## 状态责任

ADR 状态由主会话更新。执行会话不得自行将决策设为 `accepted`，但可以按任务包原样记录已由主会话批准的状态。
