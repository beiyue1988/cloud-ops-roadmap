## 4. 根因与最小修复

写前 `_mask_outline_code()` 只识别 `line.startswith(("    ", "\t"))`。该条件能处理四个字面空格或第一个字符即为 Tab，却没有按 Markdown 的 4 列 Tab stop 计算混合空格/Tab 的视觉缩进，因此三个已达到视觉列 4 的代码行仍进入受控表解析。

最小修复只调整该函数的行首识别：

1. 从列 0 开始只扫描连续的空格和 Tab；
2. 空格使列数加 1；
3. Tab 使用 `column += 4 - (column % 4)` 前进到下一个 4 列 Tab stop；
4. 首个非空格/Tab 字符终止行首扫描；
5. 视觉缩进大于等于 4 时，把完整原行交给既有 `_masked_code_line()`；否则保留完整原行；
6. 继续使用 `splitlines(keepends=True)` 和既有掩码函数，行数、字符长度和 LF/CRLF/无末尾换行状态保持；
7. 没有使用 `strip`、`lstrip`、`expandtabs`、通用 Markdown 解析器或第三方依赖。

围栏屏蔽仍先由 `_mask_fenced_code()` 完成；HTML br、显式栈依赖图、项目总视图可比较状态、OL 编号与责任、CLI、公开函数、字段和表头均未改变。
