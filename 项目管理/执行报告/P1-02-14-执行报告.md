## 8. 四层版本、运行时、CNI 与安全边界

- **概念层：** 架构、对象、控制循环、工作负载、网络、存储、安全和排障尽量版本中立。
- **主实验层：** Kubernetes 1.36.x 是唯一默认实验线；patch 从软件版本基线集中读取，章节标题和清单不复制当前 patch。
- **企业兼容层：** 1.35.x 只在存在可观察差异时作为企业兼容参考，不形成第二套完整课程。
- **历史升级层：** 1.31.14 只用于 EOL、API 迁移和逐 minor 升级风险；不作为新安装线，不直接升级到 1.36。
- **运行时：** 主组合保持 kubeadm、containerd 2.3 LTS、CRI v1、cgroup v2 和 systemd driver；kubelet 与 runtime driver 必须一致。CRI 插件禁用、端口、内核、网络、swap、代理等实际处理留在实验门。
- **CNI：** Calico 3.32 兼容结论只达到 `source-verified`；CNI 就绪前节点 NotReady 是待解释状态。NetworkManager、数据面、端口和主机防火墙必须按实际组合显式核验，不能静默关闭安全控制。
- **Pod 安全：** 不教授已移除的 PodSecurityPolicy，以 Pod Security Standards、Pod Security Admission 和其他 admission 边界为准。

Calico 3.32 当前 requirements 页面仍残留 deprecated `allow-privileged` 与 PodSecurityPolicy 文字；这部分不适用于当前 Kubernetes 安全教学。本清单仅使用该页面的 Calico 版本测试矩阵、节点和网络前置，Pod 安全范围服从 Kubernetes 当前官方 PSS/Admission 文档与任务包明确边界，因此该来源陈旧片段可消解且不构成阻塞。
