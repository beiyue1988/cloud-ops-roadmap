## 4. 根因与最小修复

### 4.1 实际 Markdown 边界

- 根因：阶段标题和所有受控表直接扫描原始 `splitlines()`，并以 `lstrip().startswith("|")` 发现表格；围栏和缩进代码没有进入大纲解析专用屏蔽链路。
- 修复：复用 fenced code 状态机并增加大纲专用缩进代码屏蔽；被屏蔽行以等长空格保留原换行和行号。阶段标题、表头、分隔和数据行都从屏蔽结果发现；真实顶层表和最多三个前导空格的表继续解析，围栏示例可被跳过。

### 4.2 HTML `br`

- 根因：两处旧正则 `r"<br\s*/?>"` 只覆盖无属性形式。
- 已实现修复：统一使用带标签名边界的 `HTML_BREAK_RE`，覆盖九项规定测试中的常规属性、大小写、额外空白和自闭合形式；`\b` 与标签闭合约束保留普通文本 `br` 和 `<bracket>` 合法对照。章节标题/目标仍由 `OL005` 负责，派生视图文本字段仍由 `OL002` 负责。
- 未关闭边界：该正则的属性体使用 `[^<>]*?`，会排除引号包裹属性值内部的尖括号，因此 `<br title="<x>">` 及其大小写/自闭合变体仍未被识别。产品提交后的主会话抽查与影响见第 7 节。

### 4.3 深依赖图

- 根因：`_catalog_cycles()` 使用递归 DFS，并用 `stack.index()` 查找回边起点，深度受 Python 调用栈限制。
- 修复：改为显式帧栈、三色状态、当前路径和位置映射；每个节点与边按确定顺序处理，不提高递归限制、不捕获 `RecursionError`，保持原有循环诊断集合与排序。

### 4.4 项目总视图诊断隔离

- 根因：R1 的 `D−V` 检查无条件执行，无法区分空集合是合法漏行还是文件、表或里程碑解析失败造成。
- 修复：总视图解析返回“里程碑集合可比较”状态；只有文件和表成功解析且所有里程碑字段可用于集合比较时才执行 `D−V`。成功解析记录的逐里程碑章节集合比较、未知锚点、重复里程碑和章节↔项目映射保持原规则。

### 4.5 README

README 增加实际 Markdown 表边界、反引号/波浪号围栏与缩进代码处理、深图确定性，以及项目 README 全部里程碑集合与贯穿项目总视图集合相等并逐里程碑章节集合相等的合同；没有扩大到教学质量或动态版本承诺。README 对“任何属性 HTML `br`”的描述高于产品提交的实际能力，需随下一轮产品返工一并纠正或由实现满足。

## 7. 主会话提交后抽查

产品提交形成后，主会话对任务包“所有属性 HTML `br` 变体必须拒绝”的边界追加了独立抽查。只读复现结果如下：

| 输入 | `HTML_BREAK_RE.search()` | 合同结论 |
|---|---:|---|
| `<br title="<x>">` | `False` | 错误通过，未关闭 |
| `<BR title="<x>" />` | `False` | 错误通过，未关闭 |
| `<br class="x">` | `True` | 常规属性形式已关闭 |
| `<bracket>` | `False` | 合法非 `br` 标签名未误报 |
