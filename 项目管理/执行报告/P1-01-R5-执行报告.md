## 5. 实现与语义边界

### 5.1 围栏语义

- opener 只允许 0–3 个前导空格；backtick opener 的 info 含 backtick 时不成立，tilde 规则独立；
- closer 必须与 opener 同字符、长度不短于 opener、只允许 0–3 个前导空格，尾部只允许空格或 Tab；
- 4 空格、Tab、混合空格加 Tab 的伪 closer 保持为围栏内容；
- 掩码继续保持字符长度、行数及 LF/CRLF/无末尾换行状态。

### 5.2 受控表与派生诊断隔离

- 受控表的数据区以空行为终止；终止前非空坏行只产生直接 OL002；
- 结构无效的表不进入项目闭合比较；
- 非法项目章节或里程碑使对应项目侧不具备派生比较资格，但不跳过其他合法记录；
- 合法且真实缺失的项目视图/映射仍产生 OL011。

### 5.3 HTML br 字面量

- 真实行内代码跨度内的 `<br>` 为字面量，不触发 OL005/OL002；
- HTML br 前连续反斜杠为奇数时视为转义字面量，为偶数或 0 时仍按真实标签拒绝；
- 行内代码 opener 若自身被反斜杠转义，不得开启代码跨度；closing run 必须与 opener run 等长；
- 非 br 标签继续允许，带属性、大小写或空白变体的真实 br 继续拒绝。

### 5.4 有界 SCC 诊断

- `_catalog_cycles()` 改为非递归、确定性的强连通分量计算；规范化后时间和空间复杂度为 O(V+E)；
- 每个多节点 SCC 或自环只返回一个成员元组，成员和分量顺序稳定；
- OL007 使用“循环分量”成员集合表达，不再用排序成员拼接不存在的箭头路径；
- 未引入 `setrecursionlimit`、`RecursionError` 兜底或第三方依赖。

README 已同步上述精确承诺，没有把实现扩大为通用 Markdown 解析器。
