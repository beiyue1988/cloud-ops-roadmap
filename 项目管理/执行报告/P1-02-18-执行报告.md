## 8. 模型工件、推理服务与容量边界

- 模型只作为运行工件和资源需求输入；章节覆盖 tokenizer、config、weights、格式、来源、许可证、校验和 remote code，但没有点名或选择模型、参数规模、量化、上下文长度、精度、并行度或显存值。
- 显存估算按 weights、KV cache、activation 和 workspace 分层，不把单一权重大小当作完整容量结论。
- vLLM 分支覆盖 engine、scheduler、worker、cache、请求生命周期、OpenAI-compatible API、continuous batching、KV cache 和并行取舍。API 协议兼容不等于模型行为完全一致。
- 服务生命周期覆盖 model load、startup、health、readiness、graceful shutdown 和恢复；服务存活不等于推理结果正确。
- 服务证据覆盖 latency、TTFT、TPOT、throughput、queue、error、overload 和 OOM，并保留 backpressure、admission、降级和容量选择门。

## 9. Kubernetes GPU、云资源与动态选择门

- Kubernetes 分支覆盖 device plugin 与 extended resource、GPU request/limit、node label、taint/toleration、affinity、topology、MIG、time-slicing 以及组件生命周期。
- 资源可见不等于设备或框架可用；共享不等于强隔离。time-slicing 不提供 MIG 的内存和故障隔离，MIG、共享模式、调度约束、计量与租户安全仍需按实际 GPU 和组件组合核定。
- GPU Operator、单独 device plugin、宿主驱动和节点生命周期是选择门，本任务没有选择 Operator、共享模式、调度器扩展、拓扑、节点规格或集群组合。
- 云上租用 GPU 且优先阿里云 GPU ECS 的平台方向保持不变。具体实例族、GPU、显存、地域、库存、价格、计费方式和模型组合仍是阶段 18 实验进入 `ready` 前的非阻塞决策门。
- 本任务没有登录购买页、询价下单、使用凭据或创建云资源；官方产品页只能证明产品入口与能力范围，不能证明执行日库存或固定价格。
