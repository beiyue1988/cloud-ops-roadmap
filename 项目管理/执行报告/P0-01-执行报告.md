## 6. 权限请求与异常

1. 沙箱禁止写入现有 `.git`，首次 `git init` 退出码 1；按工具规则请求权限后在原目录初始化成功。没有删除、移动或重建 `.git`。
2. 沙箱网络隔离导致首次 `git ls-remote origin` 退出码 1；获批后两次只读检查均退出码 0。
3. 自定义白名单检查第一次因 PowerShell 反斜杠正则转义错误退出码 1；命令未写入文件，改用字符串替换后完整检查退出码 0。
4. Apache 官方文本首次逐字对比退出码 1，定位为本地文件缺少官方文本开头的空行；通过 `apply_patch` 修正后对比退出码 0。
5. 额外的全暂存 `git diff --cached --check` 退出码 2，只命中任务开始前已存在、且本任务禁止修改的 `CODEX-PRD.md` 第 3–5 行 Markdown 硬换行尾随双空格。规定的工作树 `git diff --check` 和本任务新建文件的暂存检查均退出码 0，因此保留原规范不越权修改。
6. `git add` 提示当前 Git 配置未来可能把 LF 转为 CRLF；文件仍为 UTF-8，任务白名单不允许新增 `.gitattributes`，本任务未修改全局换行策略。
7. 部分只读 Git 命令提示无法访问用户级 `.config/git/ignore`；命令本身退出码 0，不影响暂存路径、提交或规定验证。
