# 会话交接

本目录保存阶段主会话之间的上下文与权限转移记录，使新主会话能够从版本化文件恢复项目状态，而不是依赖旧聊天记录。交接文件不替代 `AGENTS.md`、PRD、ADR、任务包、执行报告、独立审查或最终验收报告。

## 允许编辑者

只有当前获用户授权的主会话可以创建或修改交接记录。执行会话和独立审查会话不得自封为主会话，不得通过交接文件修改任务状态、扩大任务权限或替代最终验收。

## 命名与必需内容

阶段交接文件使用 `Phase-<编号>-主会话交接.md` 命名。每份记录必须包含：

- 交接双方角色与唯一主会话声明；
- 阶段状态、固定提交和权威文件入口；
- 新主会话的启动门禁与实际验证要求；
- 当前授权范围、禁止范围和未决决策门；
- 任务签发、独立审查和最终验收责任；
- 停止条件、外部操作与安全边界；
- 接任生效条件和可直接复制的启动指令。

## 状态与历史

交接文件记录上下文和领导权转移，不直接改变任务状态。任务状态仍以[任务看板](../任务看板.md)为准，阶段结论仍以[验收记录](../验收记录/README.md)为准。后续阶段应创建新的交接文件，不覆盖已经生效的历史记录。

## 安全边界

不得保存聊天全文、密码、Token、Cookie、私钥、云 AccessKey、带签名 URL、个人敏感信息或其他凭据。只记录公开入口、安全使用方法和可复验的治理事实。
