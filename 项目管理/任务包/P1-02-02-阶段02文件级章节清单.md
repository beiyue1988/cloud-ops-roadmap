5. 同网段/跨网段判断、默认网关、路由表、最长前缀匹配与逐跳转发；
6. ICMP、MTU 与常见可观察网络现象的高层定位；
7. 传统 NAT/PAT 的地址转换目标、状态和边界；
8. DHCPv4 的地址配置角色与基本租约流程；
9. DNS 的名称空间、解析器、权威服务器、递归/迭代与缓存定位；
10. 端口、套接字、UDP 与 TCP 的服务差异，以及 TCP 建连、可靠传输和关闭的基本机制；
11. HTTP 请求/响应语义与 HTTPS 中 TLS 的高层保护位置；
12. 抓包观察、分层证据和从本机到应用的基础连通性排查方法。

## 门禁 6：一手资料

核验日期均为 2026-07-15。RFC 用于协议职责与机制边界；旧但仍有基础定位的 RFC 必须结合状态页和更新关系理解，不得把历史细节无条件写成当前实践。

| 范围 | 入口 | 支持边界 |
|---|---|---|
| TCP/IP 分层、IP/ICMP/传输定位 | RFC 1122：<https://www.rfc-editor.org/rfc/rfc1122>；RFC 792：<https://www.rfc-editor.org/rfc/rfc792> | 支持链路、互联网、传输、应用分层和 ICMP 控制/错误定位；不证明所有现代实现细节 |
| ARP 与本地链路 | RFC 826：<https://www.rfc-editor.org/rfc/rfc826> | 支持协议地址到链路地址解析；不授权展开交换机配置专题 |
| IPv4 CIDR、子网和路由聚合 | RFC 4632：<https://www.rfc-editor.org/rfc/rfc4632> | 支持无类别前缀、分配与聚合；不得回退为分类地址默认教学 |
| NAT | RFC 3022：<https://www.rfc-editor.org/rfc/rfc3022> | 支持传统 NAT/NAPT 的转换模型与限制；不代表所有厂商实现 |
| DNS 与 DHCPv4 | RFC 1034：<https://www.rfc-editor.org/rfc/rfc1034>；RFC 2131：<https://www.rfc-editor.org/rfc/rfc2131> | 支持 DNS 概念/设施和 DHCP 地址配置流程；不证明当前环境配置 |
| UDP 与 TCP | RFC 768：<https://www.rfc-editor.org/rfc/rfc768>；RFC 9293：<https://www.rfc-editor.org/rfc/rfc9293> | 支持无连接数据报与 TCP 连接/可靠字节流定位；不要求实现字段级教程 |
| HTTP 与 TLS | RFC 9110：<https://www.rfc-editor.org/rfc/rfc9110>；RFC 8446：<https://www.rfc-editor.org/rfc/rfc8446> | 支持 HTTP 语义和 TLS 保护机制的高层边界；不扩展为 Web 开发或证书运维专题 |
| 抓包与证据 | Wireshark User’s Guide：<https://www.wireshark.org/docs/wsug_html_chunked/> | 支持捕获、查看与分析协议数据的工具定位；不证明任何具体网络已经验证 |
