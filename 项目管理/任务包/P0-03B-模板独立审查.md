3. `项目管理/执行报告/P0-03-执行报告.md` 已存在；

## 目标

由未参与 P0-03 模板创建的会话独立判断模板基线是否完整、自适应、可直接复制、可机械检查，并确认其没有成为第二份 PRD、AI 规则、动态版本表或正式内容集合。

## 学习者价值

防止不安全、僵化或证据不足的模板扩散到后续所有章节、实验、项目、故障、面试和图表任务中。

## 审查范围

- `模板/README.md` 和十类模板全文；
- P0-03 执行报告的证据完整性；
- 模板提交与报告提交的文件范围；
- 批准设计、实施计划和 P0-03 任务包的一致性；
- 自动复验和人工语义审查。

## 非范围

- 不修复模板；
- 不修改 P0-03 执行报告；
- 不创建正式内容、脚本、配置或图表；
- 不修改任务状态、PRD、AI 规则、版本基线、计划或任务包；
- 不执行 P0-04 或任何后续任务；
- 不执行远程写操作。

## 必须完成的人工审查

逐项给出“通过/不通过/需说明”和证据：

1. 每个模板是否职责单一且可直接复制；
2. 占位符与 HTML 注释是否清楚、互不冲突；
3. 知识模板是否真正按 `type` 自适应；
4. 知识模板是否避免机械安装、实验、故障和配图模块；
5. 实验模板是否具备安全、成本、回滚、清理和验证门禁；
6. 项目模板是否保持通用而没有泄露具体项目答案；
7. 故障模板是否证据驱动并区分恢复与永久修复；
8. 面试模板是否建立学习产物来源并阻止虚构经验；
9. 图表模板是否降低认知负担、支持可访问性和版权追踪；
10. 任务包模板是否覆盖 PRD 15.3 十一项门禁；
11. 执行报告模板是否解决报告提交哈希的循环问题；
12. 验收报告模板是否保持独立复验和最终裁决边界；
13. ADR 模板是否包含真实选项、后果和复审条件；
14. 模板之间的字段、状态、证据和日期术语是否一致；
15. 模板是否没有重复或改写 PRD、AI 规则和版本基线；
16. 模板是否没有正式教学答案、伪造验证或敏感信息；
17. README 是否准确说明模板职责、复制和变更复验；
18. 执行报告是否完整记录验证、异常、风险和真实提交。

## 问题分级与建议结论

- **阻塞：** 范围越界、治理职责倒置、安全或敏感信息风险、提交对象无法确定；
- **必须修改：** 会使后续内容机械化、证据失真、模板不可复制或关键门禁缺失；
- **建议修改：** 不阻塞模板基线启用，但可提高一致性或可维护性；
- **通过项：** 有直接文件或命令证据支持。

建议结论只能是：

- `passed`：无阻塞、无必须修改、无建议；
- `passed-with-notes`：无阻塞、无必须修改，仅有建议；
- `changes-required`：存在任一阻塞或必须修改。

审查会话无权宣布 P0-03 或 P0-03B `accepted`。

5. 18 项人工审查逐项结果与证据；

7. 执行报告与实际仓库的一致性；
8. 风险和未决问题；
9. 建议结论；
10. 最终验收权属于主会话的声明；
11. 未执行远程写操作的确认。

```text

```

禁止修改被审查模板，禁止 amend 或 rebase P0-03 提交，禁止 push、创建 PR、发布网站或修改 GitHub 设置。

## 停止条件

- 当前任务仍为 `planned` 或任务包没有真实哈希与启动指令；
- 审查会话参与过 P0-03；
- P0-03 不是 `submitted` 或 P0-03B 不是 `ready`；
- 工作区不干净、分支或远程不符；
- 模板、执行报告或两个被审查提交缺失；
- 提交范围与执行报告不一致且无法只读核实；
- 需要修改唯一报告文件以外的内容；
- 发现敏感信息；
- 同一外部阻塞连续出现三次。
