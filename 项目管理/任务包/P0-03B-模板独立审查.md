### 相对链接与 Git 检查

使用下面的不落盘只读 PowerShell 检查 `模板/` 内 Markdown 相对链接。忽略外部 URL、邮件和页内锚点；失效数必须为 0。`模板/README.md` 必须有 10 个指向十类模板的有效相对链接。

```powershell
$files = Get-ChildItem -LiteralPath '模板' -File -Filter '*.md'
$broken = @()
$relativeLinks = 0
foreach ($file in $files) {
  $text = Get-Content -LiteralPath $file.FullName -Raw -Encoding UTF8
  foreach ($match in [regex]::Matches($text,'\[[^\]]+\]\(([^)]+)\)')) {
    $target = $match.Groups[1].Value.Trim()
    if ($target -match '^(https?://|mailto:|#)') { continue }
    $relativeLinks++
    $pathOnly = ($target -split '#',2)[0]
    $resolved = [IO.Path]::GetFullPath((Join-Path $file.DirectoryName $pathOnly))
    if (-not (Test-Path -LiteralPath $resolved)) { $broken += "$($file.Name) -> $target" }
  }
}
$readme = Get-Content -LiteralPath '模板/README.md' -Raw -Encoding UTF8
$templateLinks = ([regex]::Matches($readme,'\[[^\]]+\]\(([^)]+模板\.md)\)')).Count
Write-Output "relative_links=$relativeLinks readme_template_links=$templateLinks broken=$($broken.Count)"
if ($broken.Count -ne 0 -or $templateLinks -ne 10) { $broken; exit 1 }
```

10. 任务包模板是否覆盖 PRD 15.3 十一项门禁；
11. 执行报告模板是否解决报告提交哈希的循环问题；
12. 验收报告模板是否保持独立复验和最终裁决边界；
13. ADR 模板是否包含真实选项、后果和复审条件；
14. 模板之间的字段、状态、证据和日期术语是否一致；
15. 模板是否没有重复或改写 PRD、AI 规则和版本基线；
16. 模板是否没有正式教学答案、伪造验证或敏感信息；
17. README 是否准确说明模板职责、复制和变更复验；
18. 执行报告是否完整记录验证、异常、风险和真实提交。

## 问题分级与建议结论

- **阻塞：** 范围越界、治理职责倒置、安全或敏感信息风险、提交对象无法确定；
- **必须修改：** 会使后续内容机械化、证据失真、模板不可复制或关键门禁缺失；
- **建议修改：** 不阻塞模板基线启用，但可提高一致性或可维护性；
- **通过项：** 有直接文件或命令证据支持。

建议结论只能是：

- `passed`：无阻塞、无必须修改、无建议；
- `passed-with-notes`：无阻塞、无必须修改，仅有建议；
- `changes-required`：存在任一阻塞或必须修改。

审查会话无权宣布 P0-03 或 P0-03B `accepted`。

## 审查报告要求

`项目管理/验收记录/P0-03B-独立审查报告.md` 必须包含：

1. 任务编号、日期和独立性声明；
2. P0-03 模板提交和报告提交完整哈希；
3. 自动复验命令、退出码和关键输出；
4. 文件集合、字段、占位符、版本、敏感信息和链接结果；
5. 18 项人工审查逐项结果与证据；
6. 阻塞、必须修改、建议修改和通过项分组；
7. 执行报告与实际仓库的一致性；
8. 风险和未决问题；
9. 建议结论；
10. 最终验收权属于主会话的声明；
11. 未执行远程写操作的确认。

## 提交与远程限制

审查会话只能提交独立审查报告，提交信息固定为：

```text
docs: review project content templates
```

禁止修改被审查模板，禁止 amend 或 rebase P0-03 提交，禁止 push、创建 PR、发布网站或修改 GitHub 设置。

## 停止条件

- 当前任务不是 `ready` 或任务包没有真实哈希与启动指令；
- 审查会话参与过 P0-03；
- P0-03 不是 `submitted` 或 P0-03B 不是 `ready`；
- 工作区不干净、分支或远程不符；
- 模板、执行报告或两个被审查提交缺失；
- 提交范围与执行报告不一致且无法只读核实；
- 需要修改唯一报告文件以外的内容；
- 发现敏感信息；
- 同一外部阻塞连续出现三次。
