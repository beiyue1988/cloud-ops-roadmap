## 2. 学习者价值

学习者需要把脚本和单机服务运维升级为可追溯、可评审、可批量、可复验的变更能力。Git 与 Ansible构成就业核心；运维 API 和 IaC 建立完整成长路径，但不阻塞三个月就业主线。

## 3. 阶段边界

- 阶段 08 负责 Shell/Python 工具本身；阶段 11 负责版本化、协作、批量期望状态与变更治理。
- 阶段 09/10 提供受管 Web/Java/MySQL/Redis 对象；阶段 11 不重复服务专项正文。
- 阶段 12 负责 ECS/VPC/RAM 等阿里云资源语义和具体 Provider 场景；阶段 11 只讲中立 IaC 模型。
- 阶段 15 负责 CI/CD 平台、制品库/镜像库、环境晋级和供应链；阶段 11 的制品只覆盖身份、版本、校验和与来源。
- 复杂 Git 历史改写、Ansible 插件开发、复杂 Module/Provider 开发和团队远端 State 平台实现不纳入就业核心。
