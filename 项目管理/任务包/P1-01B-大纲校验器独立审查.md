## 门禁 8：审查标准

### 8.1 规格符合性

逐项判断：

1. 默认命令未启用大纲门禁且保持现有摘要、退出码和五类规则；
2. `--outline-gate` 只接受 `partial`、`catalogs`、`views`、`complete`；
3. `partial` 不要求未进入波次的文件，但验证已经出现的受控表格；
4. `catalogs` 要求 21 阶段、合法字段、全局唯一、直接前置存在且有向无环、就业标签闭合；
5. `views` 与波次 3 文件集合一致，十二周路线集合和顺序正确；
6. `complete` 闭合检查点、实验等级、项目入口、五个项目、贯穿视图和双向映射；
7. `OL001`–`OL012` 的实际触发与任务包责任一致；
8. 不生成、不修复、不联网、不引入第三方包或中央注册表。

### 8.2 代码质量与异常边界

必须检查：

- 受控表格解析不会静默忽略额外列、重复标题、错误分隔、HTML 换行或非法列表；
- 文件读取、路径解析和遍历只捕获预期 `OSError`/`UnicodeError`/`ValueError`，不会吞掉程序缺陷；
- 图环检测、前置顺序和闭包算法对空图、链、分支和环稳定；
- 重复诊断去重不掩盖应分别定位的冲突；
- 错误路径为相对 POSIX 路径，行号真实或按合同稳定为 0；
- 未知 CLI 参数返回参数错误，不打印 traceback；
- 当前仓库规模下没有不合理的重复全仓遍历或不可控递归；
- 单文件增长虽由计划白名单决定，内部函数和数据类仍保持可理解职责；
- README 没有宣称测试无法证明的教学或动态版本结论。

### 8.3 测试质量

必须确认：

- 原 44 个测试方法仍存在且断言未被弱化；
- 新增测试数、总数和 skip 数与执行报告一致；
- 每个 `OL` 规则不是只检查任意错误，而是检查对应责任；
- 四级通过夹具包含真实规定文件和受控表；
- CLI 测试实际检查 stdout/stderr 与退出码；
- 确定性测试重复运行同一输入；
- 关键反例不会因另一个更早错误而假通过；
- 测试只使用系统临时目录并自动清理。

### 8.4 严重级别与结论

- Critical：错误接受无效依赖/映射、破坏默认校验、安全或数据风险；
- Important：遗漏合同、错误规则归属、异常被吞、测试假阳性、文档与行为不一致；
- Minor：不影响正确性的可读性、命名、重复或说明改进；
- `passed`：无 Critical/Important，命令和范围全部通过；
- `changes-required`：存在至少一个 Critical/Important；
- `blocked`：固定对象、环境或规格冲突使审查无法完成。
