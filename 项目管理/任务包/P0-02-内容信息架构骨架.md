## 内容要求

### 顶层内容 README

每个 README 至少包含：

1. 目录定位与职责；
2. 本目录保存的权威内容类型；
3. 明确非范围；
4. 文件或子目录命名规则；
5. 指向已经存在的上游与下游 README 的相对链接；
6. 当前阶段说明，明确这里只建立骨架。

`学习路线/README.md` 和 `技术索引/README.md` 必须同时包含“仅导航”“不复制正文”和指向 `知识库/README.md` 的有效相对链接。`知识库/README.md` 必须包含“唯一权威来源”或语义完全等价的明确声明，并列出 21 个阶段的有效链接。

### 21 个阶段 README

每个阶段 README 只能包含：

- 阶段编号和名称；
- 阶段目标；
- 主要主题列表；
- 前置阶段；
- 阶段产出；
- 与前后阶段及相关内容根目录的已有链接；
- 当前不含正式正文的边界声明。

阶段名称、核心范围和产出必须与 `CODEX-PRD.md` 第 3.2 节一致。阶段 00 没有前置阶段；其他阶段应给出真实的直接前置，不得简单把所有前序阶段全部列出。P1 传统项目检查点不是知识阶段，不得在 `知识库/` 下创建第 22 个阶段目录。

### 实验等级 README

- Level 1：操作和验证单个知识点；
- Level 2：组合多个知识点完成部署；
- Level 3：维护并处理故障；
- Level 4：设计方案并解释取舍。

只说明收录标准、命名规则、前置/后续关系和当前边界，不编写实验正文。

### 项目 README

五个项目必须体现同一业务的演进关系：若依传统部署 → Docker 容器化 → Kubernetes 云原生部署 → 可观测性建设；AI 推理服务作为独立高阶项目。项目 1 可引用已经确认的 RuoYi-Vue `v3.9.2`、commit `0e2d75c` 和官方仓库，但不得下载或提交上游源码，不得写部署步骤。

### 辅助目录 README

故障、面试、运维思维、企业规范、图表、脚本、参考资料和版本记录各自只描述所有权和边界。面试体系必须声明题目来源于知识正文、实验和项目，不形成脱离实践的背题库。图表目录必须区分 Draw.io 可编辑源文件与导出图；简单 Mermaid 图仍内嵌在未来 Markdown 正文中，不在本任务创建。脚本目录不得创建语言子目录或脚本文件。
