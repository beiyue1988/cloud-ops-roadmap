### 8.2 章节清单合同

阶段 README 中唯一 `## 章节清单` 下的第一张表使用精确列：

```text
章节 ID | 预定文件 | 标题 | 主要目标 | type | 直接前置 | 建议投入 | 就业标签 | 实践锚点
```

解析规则：

- ID 为反引号包裹的 `NN.CC`，阶段号匹配目录，阶段内从 `01` 连续递增，全局唯一；
- 文件为反引号包裹的 `NN.CC-章节名称.md`，无空格，与 ID 对齐且全局唯一；
- 标题和主要目标为单行非空文本，不接受表格竖线或 HTML 换行；
- `type` 使用 PRD 九种枚举；
- 直接前置为 `—` 或逗号加空格分隔的反引号 ID，只能指向已定义且排序更早的章节；
- 建议投入仅为 `30–60 分钟`、`60–90 分钟`、`90–120 分钟`；
- 就业标签仅为 `必须学`、`建议学`、`就业后补学`；
- 实践锚点为 `—` 或逗号加空格分隔的反引号 `CP-NN`、`LAB-L1`–`LAB-L4`、`PRJ-NN-MCC`。

### 8.3 四级门禁

| 门禁 | 通过条件 |
|---|---|
| `partial` | 只验证已存在的受控表格和引用，不要求尚未进入波次的规定文件存在；当前 Phase 0/1 骨架应退出 0 |
| `catalogs` | 要求 00–20 全部阶段清单存在；ID/文件唯一；直接前置存在、无前向、无自引用、无循环；就业标签闭包合法 |
| `views` | 包含 `catalogs`，并要求批准计划列出的路线与三类索引文件存在、表头合法、引用有效；十二周主线与 `必须学` 前置闭包完全相等 |
| `complete` | 包含 `views`，并要求检查点、四级实验锚点、项目入口与五个项目里程碑存在；章节和项目里程碑双向映射一致 |

### 8.4 派生视图与闭包

- 三个月路线恰好包含第 1–12 周；主线 ID 只出现一次，并等于所有 `必须学` 章节加递归前置闭包；

- `必须学` 依赖 `就业后补学` 直接失败；

## 停止条件

- 分支、origin、工作区、任务状态或固定提交不符；
- 任务包最近提交不存在或不在当前历史；
- 现有 44 项测试或默认仓库校验在写操作前失败；
- 需要新增文件、第三方包、中央注册表或修改白名单外文件；
- 已批准设计、实施计划和现有 CLI 兼容性无法同时满足；
- 需要删除、跳过、重命名或弱化现有测试才能继续；
- 同一验证失败两次且无法通过系统化调试在白名单内消解；
- 发现敏感信息、真实凭据、外部写需求或并行文件所有权冲突；
- 无法取得总数不低于 72 且全部通过的测试证据；
- 任何内容目录出现未授权正式正文或实现文件。
