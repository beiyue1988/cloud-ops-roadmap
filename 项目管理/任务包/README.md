# 任务包

本目录保存可直接交给独立执行会话的工作合同。文件使用 `<任务编号>-简短标题.md` 命名，例如 `P0-01-Git与项目治理骨架.md`。

## 允许编辑者

任务包由主会话创建和修改。执行会话只能读取，不得补写范围、扩大权限或更改验收标准。

## 必需字段

每个任务包必须包含完整启动指令、任务编号、目标与学习者价值、范围与非范围、允许文件、依赖与版本基线、必读规范、官方资料、交付物、验收标准、验证命令、停止条件和汇报格式。

## 状态责任

只有主会话能将任务标记为 `ready`、`submitted`、`accepted` 或其他正式状态。执行会话发现规格冲突时停止并提交裁决，不自行修订任务包。
