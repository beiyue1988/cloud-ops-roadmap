# 项目实战

## 目录职责

本目录只定义贯穿项目入口、演进关系、里程碑与未来内容归属。知识章节仍以 `知识库/` 为唯一正文源，项目通过稳定章节 ID 组合已有能力，不复制知识正文。

## 五个项目入口

| 项目 | 关系 | 定位 |
|---|---|---|
| [01 若依传统部署](./01-若依传统部署/README.md) | 主线起点 | 固定 RuoYi-Vue `v3.9.2` / commit `0e2d75c`，建立传统企业应用运行、发布与交接边界 |
| [02 Docker 容器化改造](./02-Docker容器化改造/README.md) | 继承项目 01 | 复用同一业务，将传统运行单元改造成镜像与 Compose 交付边界 |
| [03 Kubernetes 云原生部署](./03-Kubernetes云原生部署/README.md) | 继承项目 02 | 复用镜像与运行边界，建立 Kubernetes 编排、发布与运维证据 |
| [04 可观测性建设](./04-可观测性建设/README.md) | 继承项目 03 | 围绕同一运行对象增加日志、指标、追踪、告警与可靠性证据 |
| [05 AI 推理服务部署](./05-AI推理服务部署/README.md) | 独立项目 | 建立独立推理服务的兼容、部署、调度、观测与退出边界，不把若依改造成 AI 系统 |

## 里程碑与证据合同

- 里程碑使用 `PRJ-NN-MCC`，每个项目固定 `M01` 至 `M04`。
- 每个里程碑只引用权威阶段清单中的 2–8 个章节；同一章节在同一项目内只进入一个里程碑。
- 证据类型只使用：设计记录、部署验证、运行证据、回滚证据、观测证据、复盘记录。
- 各项目 README 拥有里程碑定义；[贯穿项目演进线](../学习路线/06-贯穿项目演进线.md)是逐行镜像视图，阶段章节表通过同一 `PRJ-*` 形成反向映射。
- “未来内容归属”只指明后续项目材料应进入的现有项目目录，不表示实现文件已经存在。

## 无实现边界

当前只建立导航和映射，不创建项目正文、部署步骤、命令、配置、代码、脚本、图表、故障答案、面试答案、云资源或环境验证结果。动态版本、组件、GPU、模型、显存、地域和价格继续由后续任务阶段门核定。
