# 项目实战

## 目录职责

本目录组织贯穿多个知识阶段的综合项目，并维护项目之间的演进关系。

## 权威内容类型

- 同一业务的主线演进：若依传统部署 → Docker 容器化改造 → Kubernetes 云原生部署 → 可观测性建设；
- 独立高阶项目：AI 推理服务部署。

未来项目说明与实现边界以各项目目录为权威归属。

## 非范围

- 不保存正式知识正文或通用实验正文；
- 不把独立 AI 推理项目并入若依业务演进主线；
- 本任务不创建项目代码、部署步骤、配置或图表。

## 命名规则

- 项目目录使用 `NN-项目名称`；
- 目录入口固定为 `README.md`；
- 技术名保留英文，名称不含空格并以短横线连接。
