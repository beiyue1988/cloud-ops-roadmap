# 面试体系

## 目录职责

本目录组织基于已学知识和已完成实践的能力检查与工程表达入口。

## 权威内容类型

未来面试题、回答结构与追问方向以本目录为权威归属；题目必须来源于知识正文、实验和项目，不形成脱离实践的背题库。

## 非范围

- 不复制知识正文、实验步骤或项目实现；
- 不编造生产经历；
- 本任务不创建面试题、答案或题库模板。

## 命名规则

- 目录入口固定为 `README.md`；
- 后续文件名按能力主题组织，不含空格并以短横线连接，技术名保留英文。

## 上游与下游

- 知识来源：[知识库](../知识库/README.md)
- 实践来源：[实验手册](../实验手册/README.md)
- 项目来源：[项目实战](../项目实战/README.md)
- 案例来源：[故障案例](../故障案例/README.md)

## 当前阶段

Phase 0 仅建立题目来源、内容归属与导航，不含面试题或答案。
